Your data. Your rules.
AI agents play by them.

PersonalDataHub is an open-source gateway between your personal data services and AI agents. Connect Gmail, GitHub, and more — with fine-grained access control, action approval, and full audit trails. Runs locally, zero data sent to third parties.

On-the-Fly Fetching

Outbound Control

Full Audit Trail

Every data access and action is logged with a timestamp, source, and purpose string.

How It Works

Four simple steps

Set up once, control everything from the web dashboard.

Connect accounts

Link Gmail, GitHub, and other services via OAuth2. Tokens are stored locally and encrypted with AES-256-GCM.

Configure filters

Set date ranges, sender rules, subject filters, and hidden fields. Toggle them on and off from the dashboard.

Agents discover tools

Agents connect via MCP, OpenClaw, or REST API. They only see tools for sources you've connected.

Review & approve

Every outbound action enters a staging queue. You review, edit if needed, then approve or reject.

Architecture

Three layers of access control

Defense in depth — each layer independently restricts what agents can access.

1

Credential Scope

The agent holds a scoped identity that can't access resources outside the configured boundary.

2

Query Boundary

Connectors refuse to fetch data outside configured limits — date ranges, repo lists, label filters.

3

Quick Filters

Further restrict which rows are visible and which fields are delivered to the agent at read time.

Security

Built with security at every layer

Process isolation, encrypted storage, and no destructive endpoints.

🔒 What agents cannot do

  • Access data outside the configured boundary (date range, repos, labels)
  • See fields hidden by quick filters (e.g., email body can be stripped)
  • See rows excluded by active filters
  • Send emails or execute actions without owner approval
  • Delete anything — no destructive endpoints exist

🛡 How it's enforced

  • Runs as a dedicated OS user (personaldatahub) with file permissions 0600
  • OAuth tokens encrypted with AES-256-GCM
  • GUI protected with bcrypt-hashed passwords
  • Server binds to 127.0.0.1 only — no network exposure
  • Immutable audit log records every data movement

⚠ Known limitations

  • Once data passes filters to agent, PersonalDataHub can't control what the agent does with it
  • Host-level compromise yields access to OAuth tokens and encryption keys
  • No built-in rate limiting yet (future enhancement)

Get Started

Up and running in minutes

macOS example below. See the full Setup Guide for Linux instructions.

  Terminal 
# 1. Create the personaldatahub OS user (one-time, macOS)
sudo sysadminctl -addUser personaldatahub -shell /bin/zsh -password -
sudo mkdir -p /Users/personaldatahub && sudo chown personaldatahub:staff /Users/personaldatahub

# 2. Install and start (as personaldatahub)
sudo -u personaldatahub -i
cd ~ && git clone https://github.com/AISmithLab/PersonalDataHub.git
cd PersonalDataHub && pnpm install && pnpm build
npx pdh init    # save the owner password it prints
npx pdh start
exit

# 3. Configure your main user
mkdir -p ~/.pdh
echo '{"hubUrl":"http://localhost:3000","hubDir":"/Users/personaldatahub/PersonalDataHub"}' > ~/.pdh/config.json

Claude Code

"command": "npx", "args": ["pdh", "mcp"]
